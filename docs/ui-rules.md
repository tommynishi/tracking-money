# UI規約（ui-rules.md）

Tracking Money のUI実装規約です。画面ごとの設計は docs/screen.md を正とします。

---

# 1. 基本方針

* Tailwind CSS を使用する（独自CSSファイル・CSS-in-JSは原則使わない）
* PCは表形式、スマートフォンはカード形式を基本とする（CLAUDE.md）
* すべての画面でレスポンシブ・ダークモードに対応する
* 家計簿アプリとして「数字が読みやすいこと」を最優先とする

---

# 2. レスポンシブ

| 項目 | 規則 |
| --- | --- |
| ブレークポイント | Tailwind標準を使用。`md`（768px）未満をスマートフォンレイアウトとする |
| 設計順序 | モバイルファースト（無印クラス＝スマホ、`md:` 以上でPC向けを上書き） |
| 一覧表示 | PC：`<table>`／スマホ：カードリスト。同一データソース・同一コンポーネントから出し分ける |
| タップ領域 | 44px × 44px 以上を確保する |
| 横スクロール | ページ全体の横スクロールを発生させない（表は必要に応じてコンテナ内スクロール） |

---

# 3. ダークモード

* Tailwind の `dark:` バリアント（class戦略）で実装する
* 初期値は OSの `prefers-color-scheme` に従い、ヘッダーのアカウントメニューから手動切替できる（選択は localStorage に保存）
* 色はセマンティックトークン（§4）経由で使用し、`dark:` の個別指定が散らばらないようにする

---

# 4. カラー

Tailwind テーマにセマンティックトークンを定義し、**生のカラーパレット（`gray-500` 等）を画面コードへ直接書かない**。

| トークン | 用途 |
| --- | --- |
| `background` / `surface` | 画面背景／カード・モーダル背景 |
| `foreground` / `muted` | 本文／補助テキスト |
| `border` | 罫線・区切り |
| `primary` | 主要アクション（登録・確定ボタン等） |
| `danger` | 破壊的アクション・エラー表示 |
| `warning` | 警告（重複候補・Drive保存失敗等） |
| `success` | 成功表示 |
| `refund` | 返金（マイナス金額）の表示 |

* 色だけで意味を伝えない（アイコン・ラベルを併記する）
* コントラスト比は WCAG AA（本文 4.5:1 以上）を満たす

---

# 5. タイポグラフィ・金額表示

| 項目 | 規則 |
| --- | --- |
| フォント | システムフォントスタック（日本語考慮）。Webフォントは使用しない |
| 本文サイズ | `text-sm`（14px）〜 `text-base`（16px）。フォームの入力は16px以上（iOSのズーム防止） |
| 金額 | 3桁区切り＋`円`（例：`12,480円`）。等幅数字（`tabular-nums`）で桁を揃える |
| 返金（負値） | `refund` トークン色＋`−`記号（例：`−3,980円`） |
| 日付 | 一覧は `M/D（曜）`、詳細・フォームは `YYYY/MM/DD` |

金額・日付のフォーマットは `shared/utils/format` に集約し、画面ごとに実装しない。

---

# 6. コンポーネント規約

共通コンポーネントは `shared/components/` に置き、feature側で再実装しない。

| コンポーネント | 規則 |
| --- | --- |
| Button | variant：`primary` / `secondary` / `danger` / `ghost`。処理中は disabled＋スピナー表示（二重送信防止） |
| 入力系（Input / Select / DatePicker） | label 必須。エラーは項目直下に赤字表示。`aria-invalid` / `aria-describedby` を設定 |
| Modal / Dialog | フォーカストラップ・Escで閉じる・背景クリック挙動を統一。破壊的操作の確認は Dialog を必須とする |
| Table / CardList | 一覧表示の出し分けペア。ソート状態を `aria-sort` で表現 |
| Toast | 操作の成功・失敗のフィードバック。自動消去（成功4秒）・エラーは手動で閉じる |
| Skeleton | ローディング表示の既定。レイアウトシフトを起こさない |
| EmptyState | 0件表示。説明＋次アクションのボタンをセットにする |
| Badge | カテゴリ・取込元・重複・固定費などの状態表示 |

---

# 7. フォーム

* 送信中はボタンを disabled にし、多重送信を防ぐ
* バリデーションは zod スキーマをクライアント・サーバーで共有する（メッセージは日本語）
* 必須項目はラベルに「必須」バッジで明示する（`*` のみは不可）
* 入力エラー時は最初のエラー項目へフォーカスを移動する
* 離脱時に未保存の変更がある場合は確認する（明細フォーム・マッピング作成）

---

# 8. アクセシビリティ

* すべての操作をキーボードのみで完結できる（Tab順序は視覚順と一致させる）
* インタラクティブ要素はネイティブ要素（`button` / `a` / `input`）を使う（`div` に onClick を付けない）
* 画像・アイコンボタンには代替テキスト（`aria-label`）を付ける
* グラフには数値の代替表現（表・ラベル）を用意する
* モーダルは `role="dialog"` ＋ `aria-modal` ＋ フォーカス管理を実装する（共通Modalで担保）

---

# 9. グラフ（分析画面）

* カテゴリ別内訳・推移などのグラフはライブラリを使用する（採用ライブラリは実装時に選定・提案する。候補：Recharts）
* 凡例・数値ラベルを必ず表示し、色のみで区別しない
* データ0件時はグラフを描画せず EmptyState を表示する
* ダークモードでもコントラストを維持する（セマンティックトークンから配色する）

---

# 10. 画像・アイコン

* アイコンは1つのアイコンセットに統一する（採用ライブラリは実装時に選定・提案する。候補：lucide-react）
* 画像は `next/image` を使用し、遅延読み込みを既定とする
